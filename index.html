Rogue Neo

Here we have my third-year project called Rogue Neo, it's a stealth game with a strong focus on AI and behavior trees. The game is set in a world where the player is trying to steal data from a corporation. The game is made in Unreal Engine 4 and uses C++ and Blueprints. The AI enemy possess hearing and vision senses making it crucial for players to be aware of their surroundings and plan accordingly. The player was able to use distractions to create openings in order to help them create different strategies, and multiple hiding objects were placed around the level in order to keep the player hidden, the stealth design was incorporated to challenge the player, and give the game a sense of unpredictability. Rogue Neo was my one of my favourite ambitious projects that allowed me to gain knowledge on multiple areas of game development such as AI programming, level design, user testing and balancing.

Movie Database App

This is another project I've worked on unrelated to games. A mobile application using Dart, where I used an API to retrieve and display movie/tv show information in a user friendly list. By utilising the TMDB API, the app was able to dynamically fetch movie information that was kept up to date. The user can then click on the movie to get more information and also add them to a watchlist. The app featured SQLite, which is a lightweight adn efficient database that stores information locally, this was used to store the watchlist which enchanced the practicality of the application. The app also featured a search function and a sort function which allowed users to search for movies but also allow users to sort movies by popularity and alphabetically. This was done by using the API to fetch the data of the application giving it more use. Error handling was also added where appropriate to ensure that the app would funVtion correctly with even if there were problems with the API.

Zyzz

Here is a game I made during my second year of university in a team setting. We used a Kanban approach to organize and set our targets, by using this approach our tasks were able to be tracked and monitored by other members and we were able to set ourselves appropriate deadlines to meet. The game was a 3D platformer made in Unreal Engine 4 using Blueprints and some C++, we went for a blueprint approach to familiarise ourselves with Unreal Engine's features, which would help in future projects where we would be able to determine where to use blueprints when appropriate, where the player would play as a wizard navigating through levels using parkour i.e., double jump and wall jumping, to get through the levels, collecting coins, and defeat dragons along the way. The game featured level traveling, a hub area, a combat system that was able to track damage and change the values accordingly and multiple levels with different themes. Zyzz was one of the first projects where i worked in a team and organised the team's deadlines and timetable to meet deadlines, on top of developing my parts of zyzz, which was the tutorial level, level traveling system, hub world and combat system. I was also updating the Kanban, communicating with the team ensuring that deadline would be met, altering the deadlines when necessary, consulting with the team when changes were to be made and assisting the team when it came to challenging features.

Post Processing

In this project, I was introduced to post-processing effects. This involved rendering a scene to a texture instead of the back buffer, creating effects, and then copying the textures to the backbuffer to show on the screen. This project involved working with pixel shaders and vertex shaders using HLSL.
